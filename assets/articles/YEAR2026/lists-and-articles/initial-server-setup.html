The Architecture Shift

Traditional network policies often fall short in high-churn environments. With Cilium and eBPF, the security logic is embedded directly into the kernel, allowing for identity-aware enforcement.

Note: Ensure your Linux Kernel version is 5.10+ to support the latest eBPF helper functions used in this log.

Implementation: Network Policies

To restrict traffic between the frontend and backend namespaces, apply the following manifest:

network-policy.yaml 
apiVersion: "cilium.io/v2"
kind: CiliumNetworkPolicy
metadata:
  name: "secure-backend-access"
spec:
  endpointSelector:
    matchLabels:
      app: backend
  ingress:
  - fromEndpoints:
    - matchLabels:
        app: frontend
    toPorts:
    - ports:
      - port: "8080"
        protocol: TCP

Verification via CLI

Execute the following command to verify the status of the eBPF maps and verify endpoint security status:

Terminal 
$ cilium status --wait
$ cilium bpf endpoint list

Conclusion
